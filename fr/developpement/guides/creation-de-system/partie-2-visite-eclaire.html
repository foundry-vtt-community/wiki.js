Partie 2 - Visite éclair

 

Avant d'aller plus loin, nous allons faire un bref détour pour examiner ce que nous allons exactement détruire de manière catastrophique dans nos tentatives de créer un système fonctionnel. Voici les fichiers sur lesquels nous allons principalement travailler :

./template.json

Le fichier template.json contient la structure de données que Foundry applique automatiquement chaque fois qu'une entité est créée. Une entité, pour ce qui nous concerne, est simplement un personnage ou un objet qui doit avoir une fiche (formulaire) contenant des données. Foundry a d'autres types d'entités, mais nous nous intéressons uniquement aux personnages et aux objets.

 

./styles/simple.css

Le principal fichier CSS qui contrôle l'apparence de tout dans notre système. Nous nous intéressons principalement à cela pour rendre nos pages HTML plus attrayantes.

 

./templates/actor-sheet.html

Chaque fois que Foundry crée un personnage pour notre système, il utilise cette page. C'est là que nous allons créer notre fiche de personnage et la modifier pour qu'elle utilise les données que nous voulons.

 

./module/actor-sheet.js

La classe JavaScript "ActorSheet" pour notre système. Grande et effrayante. Nous n'y passerons pas beaucoup de temps.

 

./templates/item-sheet.html

Identique à actor-sheet.html, mais pour les objets. Nous ne le modifierons pas dans ce guide.

 

./module/item-sheet.js

Identique à actor-sheet.js... mais pour les objets. Vous avez compris l'idée.
